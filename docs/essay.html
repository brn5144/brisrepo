The Contrast Between GitHub and Sites at Penn State

Coming to the completion of both websites on Github and Sites at Penn State, many conclusions have been drawn. The difference in complexity and intricacy can very clearly be noticed between the two. This report will go into detail about those differences.

Between working with both sites, the most significant challenges for a new learner would stem from the GitHub site. This is for a couple of reasons. HTML code is not particularly hard to work with, but the CSS styling can be tricky because of how many different codes there are for different styling aspects. The concept of having to create an entire other page for styling is also a cause for concern - whereas in the Sites at Penn State - it is very easy to click customize and be able to visually see the customization.

The Sites at Penn State and GitHub both have their personal advantages. For example, Sites at Penn State allows for you to be able to see your site while creating it, although with GitHub, you can only write code and continuously test it. Modifying the layout with Sites at Penn State was quite easy, but lacked personalization. There were limited choices for the layout. With GitHub, it is much harder to control the layout, but you can almost fully customize it to your liking once you get the hang of it.
